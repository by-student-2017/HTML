TDDFT (TDDFPT) or EELS calculation
--------------------------------------------------------------------------------
■ Usage

◇ Examples (e.g. CH4-DAV-B3LYP)
1. cd $HOME/q-e-qe-6.3
2. cd TDDFPT
3. cd Examples
4. cd CH4-DAV-B3LYP
5. gedit Makefile
  NAME = XXXX
6. rename XXXX at following input files,
  -----Davidson
  XXXX.pw-in
  XXXX.dav-in
  XXXX.tddfpt_pp-in (sometimes)
  -----
  -----Lanczos
  XXXX.tddfpt-in
  XXXX.tddfpt-eels-in
  XXXX.tddfpt_pp-in
　-----
  etc
7. gedit input files
  e.g. gedit XXXX.pw-in, etc
  change crystal data in XXXX.pw-in.
8. make
  you can get XXXX.plot and XXXX.eigen.
※ makeでエラーが出力された場合は、make cleanで一度ファイル類を綺麗にしてから再度makeを行う

plot
1. gnuplot
2. plot "XXXX.plot" w l

3. cd Examples
4. cd CH4-DAV-B3LYP
5. mpirun -np 2 $HOME/q-e-qe-6.3/bin/pw.x < CH4.pw-in
----------davidson
6. mpirun -np 2 $HOME/q-e-qe-6.3/TDDFPT/bin/turbo_davidson.x < CH4.dav-in
  you can get CH4.plot and CH4.eigen.
----------lanczos
6. mpirun -np 2 $HOME/q-e-qe-6.3/TDDFPT/bin/turbo_lanczos.x < CH4.tddfpt-in
7. mpirun -np 2 $HOME/q-e-qe-6.3/TDDFPT/bin/turbo_spectrum.x < CH4.tddfpt_pp-in
----------eels (Examples/EELS-Ag-USPP-GGA)
6. mpirun -np 2 $HOME/q-e-qe-6.3/TDDFPT/bin/turbo_eels.x < Ag.tddfpt-eels-in
7. mpirun -np 2 $HOME/q-e-qe-6.3/TDDFPT/bin/turbo_spectrum.x < Ag.tddfpt_pp-in
※ このように入力しても動作する。上の動画の手法で表示されているコマンドと対応していることが分かる。

※ Youtubeの動画では、Davidsonのアルゴリズムを使ったファイル名にDAVの記載のある計算例で計算していますが、DabidsonではなくDAVの記載のない方法であるLanczosの方が入力ファイルでのパラメータの設定は簡単です。

※ ExamplesやREADMEを読んで計算したい手法を選択すれば良い。ノルム保存擬ポテンシャル（NCPP）以外にもウルトラソフト擬ポテンシャル（USPP）やEELSの計算やスピン軌道相互作用を含めた計算、他の交換相関ポテンシャル（TDHFやPBE0、EXX）、電荷応答、結合様式を描画する、などの例が準備されている。AgやAuなど原子番号50近傍以降ではスピン軌道相互作用が重要になる。

※ TDDFPTフォルダのコンパイルはPWscfでのコンパイル「make all」でされていたので、TDDFPTのフォルダで改めでコンパイルする必要はない。

◇ *.dav-in (davidson法を用いるものでパラメータの設定がより複雑)
　doc の INPUT_Davidson.txt を見るとよい
・ d0psi_rs
　.true. では実空間で双極子が計算される。系が有限なら実空間での双極子の取り扱いのみが許される。
　.false.では逆格子空間で双極子が計算される。
　分子は.true.、結晶は.false.を使えばよい。
　ディフォルト： .false.
・ ecutfock
　EXX（非局所厳密交換）のためのカットオフエネルギーの設定。ディフォルトでは、pw.inでのecutrhoが使われるが、ecutfockの値を減少させて計算精度を犠牲にすることで計算速度を非常に速くさせることができる。ecutwfcよりも小さくしてはいけない
　ディフォルト： ecutrho
・ num_eign
　計算する固有値の数。
　ディフォルト： 1
　※ 計算したいスペクトルの範囲にも依存するので決まった値はないが、5, 10, 15と大きくしていってスペクトルが大きく変化しなくなる値を採用する。H2Oでは5、CH4では10、Benzeneでは5。
・ num_init
　通常、num_eignの約2倍を設定する
　ディフォルト： 2
・ num_basis_max
　部分群で許される最大の基底ベクトル（線形独立なベクトル）の数
　ダビッドソンアルゴリズムで要求されるメモリ（計算が走る時に表示されるメモリの評価）
　ディフォルト： 20
　※ num_eignで大きな値を設定するほど、num_basis_maxの値も大きくなる。このパラメータの典型的な値については、TDDFPTのExamplesを参考にしてください。H2Oでは40、CH4では80、Benzeneでは120であった。
・ residue_conv_thr
　収束のしきい値
　ディフォルト： 1.0E-4
・ start
　出力されるスペクトルの開始（Ry）
　ディフォルト： 0.0
・ finish
　出力されるスペクトルの終了（Ry）
　ディフォルト： 1.0 Ry = 13.6058 eV
・ step
　出力されるスペクトルのステップ（Ry）
　ディフォルト値： 0.001
・ broadening
　ローレンツ関数でのブロードニング（Ry）
　ディフォルト値： 0.005
・ reference
　ディフォルト値： 0.0
・ p_nbnd_occ
　占有準位のバンド数
　ディフォルト値： 10
　pw.outに一番最後の方に記載されている highest occupied より負の値の固有値の数を入力すればよい
・ p_nbnd_virt
　非占有準位のバンド数
　ディフォルト値： 10
　pw.outに一番最後の方に記載されている lowest unoccupied より正の値の固有値の数を入力すればよい
・ poor_of_ram
　十分なRAMメモリがないときに.true.を用いる。.false.では2倍のRAMメモリを使うが、計算速度を増加させる。
　ディフォルト： .false.
・ poor_of_ram2
　十分なRAMメモリがないときに.true.を用いる。.false.では2倍のRAMメモリを使うが、計算速度を増加させる。
　ディフォルト： .false.
※ 基本的には、TDDFTを計算する範囲（占有準位と非占有準位）とスペクトルを描くための条件となる。

◇ *.tddfpt-in
・ itermax
　ランチョス法での繰り返しの数
　ディフォルト： 500
・ pseudo_hermitian
　.true.のときは擬エルミートランチョスアルゴリズムが用いられる。
　.false.のときは非エルミートランチョス双直交化アルゴリズムが用いられる（これは2倍遅い）。
　ディフォルト： .true.
・ ipol
　1: alpha_xx (omega)
　2: alpha_yy (omega)
　3: alpha_zz (omega)
　4: 全ての分極率テンソルと吸収係数
　ディフォルト： 1
・ n_ipol
　与えられた分極の方向について計算されるために、ゼータ係数の数を決定する
　ディフォルト： ipolが4より小さいとき1, ipolが4のとき3
・ d0psi_rs
　.true. では実空間で双極子が計算される。系が有限なら実空間での双極子の取り扱いのみが許される。
　.false.では逆格子空間で双極子が計算される。
　分子は.true.、結晶は.false.を使えばよい。
　ディフォルト： .false.
・ ecutfock
　EXX（非局所厳密交換）のためのカットオフエネルギーの設定。ディフォルトでは、pw.inでのecutrhoが使われるが、ecutfockの値を減少させて計算精度を犠牲にすることで計算速度を非常に速くさせることができる。ecutwfcよりも小さくしてはいけない
　ディフォルト： ecutrho

◇ *.tddfpt-eels-in
・ itermax
　ランチョス法での繰り返しの数
　ディフォルト： 500
・ q1, q2, q3
　直行座標での移行運動量の値、単位は[2π/格子パラメーター]
　ディフォルト： 1
・ pseudo_hermitian
　.true.のときは擬エルミートランチョスアルゴリズムが用いられる。
　.false.のときは非エルミートランチョス双直交化アルゴリズムが用いられる（これは2倍遅い）。
　ディフォルト： .true.
　※ 何らかの理由でうまくいかない限り、常に.TRUEを使用することをお勧めします。

◇ *.tddfpt_pp-in
・ eels
　.true.では EELSスペクトルが計算される
　ディフォルト： .false.
・ td
  lanczos: ランチョスの係数を用いてスペクトルの計算がされる
  david: ダビッドソンアルゴリズムで固有値を用いてスペクトルが計算される
　davidson: ダビッドソンアルゴリズムで固有値を用いてスペクトルが計算される
　ディフォルト： lanczos
・ itermax0
　ファイルから読み込まれるランチョス係数の数
　※ 通常、itermax と同じ値を設定する
・ itermax
　ランチョス係数の合計数
　ディフォルト： 500
・ extrapolation
　外挿の設定。
　osc: biconstant extrapolation
　constant: constant extrapolation
　no: no extrapolation
　ディフォルト： no
・ epsil
　ブロードニング / ダンピング項 (Ry単位）
　ディフォルト： 0.02
・ units
　0 では Ry
　1 では eV
　2 では nm/eV
　ディフォルト： 0 で、Ry単位となる
・ start
　出力するスペクトルを開始するエネルギー
　ディフォルト： 0
・ end
　出力するスペクトルを終了するエネルギー
　ディフォルト： 2.5
・ increment
　エネルギー刻み
　ディフォルト： 0.001
・ ipol
　1: alpha_xx (omega)
　2: alpha_yy (omega)
　3: alpha_zz (omega)
　4: 全ての分極率テンソルと吸収係数
　ディフォルト：1
・ eign_file
　ディフォルト： pwscf.eigen
　※ turbo_davidson.xで作成されたファイルの名称を記入する。そのファイルの中には固有値が記載されている。

◇ *.tddfpt-in-s2 and *.tddfpt-in-s2-2
・ itermax
　ランチョス法での繰り返しの数
　ディフォルト： 500
・ ipol
　1: alpha_xx (omega)
　2: alpha_yy (omega)
　3: alpha_zz (omega)
　4: 全ての分極率テンソルと吸収係数
　ディフォルト：1
・ charge_response
　1とすると電荷密度の応答を出力する
　ディフォルト： 0
・ project
　※ マニュアルに記載無し。Examplesでは.true.が設定されている。
・ omeg
　この遷移エネルギーに対して電荷密度の応答が計算される（Ry単位）
　ディフォルト： 0.0
・ epsil
　ブロードニング / ダンピング項 (Ry単位）
　ディフォルト： 0.0 (スペクトルでのepsilとはディフォルトの値が異なるので注意）
・ w_T_npol
　前の計算で考慮した分極の方向の数
　ipol = 4 なら w_T_npol = 3 にしなければならない。他ではw_T_npol = 1にしなければならない
　ディフォルト： 1
・ plot_type
　1: オリジナルの出力
　2: Xcrysden 用の出力
    vesta *.xsf （←私のPCの環境では失敗）
    xcrysden --xsf *.xsf （←私のPCの環境では失敗）
　3: gaussian cube 用の出力
    vesta *.cube
    xcrysden --g98_cube *.cube （←私のPCの環境では失敗）
　ディフォルト： 1
　※ plot_type=3 として出力された*.cubeファイルをvestaで開くのが良い。
　※ 恐らく、ipol=4だと、 1: alpha_xx (omega)、2: alpha_yy (omega)、3: alpha_zz (omega)に対応して、adsorbtive-polが1から3まで, dispersive-polが1から3まで出力される。
　※ CH4-BONDではomegの値を変えてこの振る舞いの変化を見ている

----------reply----------
Dear X. XXXXX
I will try to reply your questions:

1. num_eign = number of eigenstates (excitations) which will be computed. This parameter must be chosen based on how large energy range for the spectrum the user is interested in. The bigger the energy range, the larger num_eign is. A priori it is not possible to guess this number, because it depends on the system. Try with some reasonable numbers (say, 5, 10, or 15) and then check whether this is enough to obtain a converged spectrum in the energy range of interest.

2 'num_basis_max' is the max number of basis allowed in the subspace, every davidson iteration the algorithm increases about of num_init the number of basis used in subspace. When reach num_basis_max discharging routine is called, this variable determines the memory requirement. the larger the parameter num_eign, the large num_basis_max would be needed. Try with some reasonable numbers, run test calculations, and then increase/decrease num_basis_max based on your observations. Use examples in TDDFPT to get an idea about typical values of this parameter.

3. 'ecutfock' is the kinetic energy cutoff (Ry) for the exact exchange operator in the exact-exchange (EXX) type calculations. The meaning of this parameter is the same as in the PW code (pw.x), hence see the documentation in PW/Doc. The value of ecutfock cannot be smaller than ecutwfc.

4. 'itermax0' generally you can use the same number that you computed with turbo_laczos (itermax)

5. 'p_nbnd_occ and p_nbnd_virt' if you are interested only in energy and absorption spectrum, is not necessary use all occupied and virtual orbitals. Using all states is useful only if you are interested in the interpretation of the eigenstates.

6. 'pseudo_hermitian' When you use .FALSE., the code will be 2 times slower but the result must be the same. So it is always recommended to use .TRUE. unless it does not work for some reasons. The .FALSE. option was implemented first, and hence it is still kept in the code for the case when .TRUE. does not work.

7. 'CH4-BOND' this is obsolete and will be removed in the next release of QE. The meaning of "project", "absorbtive", and dispersive" is not documented, it was introduced by Osman Baris Malcioglu around ~2011 (CPC 182, 1744 (2011)). These parameters are related to the calculation of the response charge density. You may try either to understand what does these keywords mean by looking into the code or try to contact Baris and ask him directly.

Best regards.
Oscar Baseggio
