■ 基本的な情報
  原子番号が50を超える原子を扱う場合は相対論版が良い。
A1. 相対論版DV-Xa法 (相対論版 dvxar)
A2. 相対論版DVME法
　相対論版DVME法のプログラムを利用したい場合は、関西学院大学の小笠原教授に共同研究を申し込めば良いそうです。ご興味のある方は検討してみてください。
B. 遷移確率計算プログラム(sxs及びsxsplt)（相対論版であるrsxsは開発者に相談）
C. 全エネルギー計算プログラム(TESDA及びCoulomb)（開発者に相談）
D. 対称軌道作成プログラム(makef25)
E. 配置間相互作用を考慮したプログラム(DVME法)[DMEX1]
F. CVV Auger スペクトルの計算プログラム（入手可能かは不明）
G. 光電子スペクトル計算プログラム(PES)
※ Linux PCの方が性能が良い場合は、wineをインストールして、下記と同様にすれば良い。
　wineの起動コマンドは[wine cmd]となる。$HOME/.wine/drive_c が主に使用するディレクトリになる。
※ PESはGeliusの方法を用いている。各原子軌道成分の部分状態密度に、原子軌道のイオン化確率（分子軌道計算に用いた原子軌道に対するイオン化断面積をDVXa法で計算）をかけたものの和で近似する。

■ セットアップ（Windows）
　『新版 はじめての電子状態計算』に記載されているようにデータを配置する。
1. ダウンロードしたファイルを解凍する
　私の場合はLhaplus（http://www.vector.co.jp/soft/dl/win95/util/se169348.html）を用いている
2. dvxa_v2.zipを解凍するとdvxa_v2のフォルダができる
  C:\Users\ユーザー名にdvxa_v2の中にあるdvxaを移す
  私の場合は「C:\Users\ユーザー名」が 「C:\Users\Owner」
3. dvxaフォルダの中にある exec の DvMenu.exe をクリック
4. DisplatDVをクリック
　下記のような（×）を伴ったエラーが出た場合は下記の5.に続く。
　
5. Visual Basic 6.0（http://www.vector.co.jp/soft/dl/win95/util/se342080.html）をダウンロード
  ダウンロードしたvb6rt340.exeをクリックするとvb6rtestのフォルダができる
  setup.exeをクリック（基本的に「OK」と「はい」を選択）
  ※ 例えば、私の場合は OK > PCが描かれたアイコン > 継続 > はい > はい > OK をクリックしていった。
  ※ 私は再度インストールなどと表示されたらそれを選択した。
6. DisplatDVをクリックして（×）を伴ったエラーが出なければ終了

■ 入力ファイルの作り方
1. DvMenu.exe > DisplatDV
2. [DisplatDV]
    設定 > 座標入力 > CIFにチェックを入れる> File Open > 読み込むcifファイルを選択
3. [Display Structure]
    Cell Size > X: と Y: と Z:で拡張する倍率を入力（[-1 -1]のアイコンをクリックするのがよい） > On Edgeにチェックを入れる > Expand > OK
    Cellにチェックを入れる（何度も押すと囲まれる範囲が変わる）
4. [DisplatDV]
    座標の表示のDを何度かクリックして、原点からの距離長さの順にする
    中心付近にある分析したい原子をクリック > 原点をクリック
    ※ [Display Structure]でMouseのon Listにチェックを入れて直接画像から中心となる原子を選択 > [DisplatDV]の原点をクリックでも可能
5. [Display Structure]
    Shift の隣にあるバーで表示する原子の数を変えられる > Del. Outer Atomでカットできる
6. [DisplatDV]
    DV関係 > F01の作成
7. F01の作成
    Allにチェックを入れる > Start symchk > 左下に表示されている--OKが付いたもので一番下のものを選択（下に行くほど対称性が高い）> F01 setting > Check > Charge: で形式電荷を入力 > F01 > OK
8. [DisplatDV]
    DV関係 > F03の作成
9. F03の作成（イオン結合性が強い化合物）
    Start > 原子を指定してChargeを選択 > Save F03 > OK
※ 新版第一印刷 正誤表：p.113, [結晶操作] -> [DV関係]

□ 入力ファイル（クラスター）作成のポイント
1. 共有結合性の強い元素を中心に配置
　SiやGeなどの共有結合が元素が一番外側になると（ダングリングボンドが形成される影響で）ギャップ中に準位が現れるので水素で終端するとよい。水素終端はWinmoster（2017年ではアカデミックは無料）を用いればよい
2. 酸化物などは金属原子に配位している酸素原子は全て考慮する方がエネルギー順位などの計算精度がよくなる場合がある
3. 酸化物のようなイオン結合性が強い化合物はマーデルング・ポテンシャルを考慮するとよい
4. 金属ガラスなどでは、中心からの原子の距離Dの分布を見ると、Dの分布で大きなギャップが開いているところがある。そこを境にしてクラスターを切り取るとよい

----------日本語
通常用いるのはmdstとsig1です．
  mdst=0           GB，FDオフ
      =1            Gaussian Broadening
      =2                     Fermi-Drac Distribution
  sig1         GBの時は半値幅の1/2（正規分布のσ値）
                     FDの時は温度（K）
文献等ではGBの時0.05〜0.1 eV程度が用いられるようですが，0.01eVでも効果はあります．また，GB/FDを使うと安定して収束に向かうようになります．すなわち，収束に向かう軌道の割合（f06zの左から三つ目の値）が高くなります．通常の設定では収束に向かう軌道の割合が60%未満になったときにMixingを小さくするようになっていますが，GB/FDを使う場合は，この設定値（fcvj）をもう少し大きくしたほうが，収束が早くなる場合があります．
引用文献：DV2.doc; http://chem.sci.hyogo-u.ac.jp/hajimete/download.html
----------English
ncycm = number of cycle
cvj = convergence parameter (Please, change cvj to more large value by using of many atoms, e.g. cvj = 0.01 -0.001 at 150 atoms)
mdst: 0 = off, 1 = Gaussian Broadening (GB), 2 = Fermi-Drac Distribution (FD)
sig1: half width * 1/2 for GB (sigma of Gaussian distribution), Kelvin for FD
Papers often use 0.05 - 0.10 eV for GB, but there is effect by using of 0.01 eV.
fcvj: change 0.6 to 0.7, 0.7 setting is faster than 0.6.
tdxc1 less than tdxc: use sig1
tdxc2 less than tdxc less than and equal to tdxc1: use sig2
tdxc less than and equal to tdxc2: use sig3
tdxc = The amount of movement of the charge (value on the left in f06z)
Can not recommend setting sig2, sig3, tdxc1, tdxc2, because population change might occuers.

■ 結果の解釈の前に
『新版 はじめての電子状態計算』では非常に多くの機能が盛り込まれた。しかしながら、計算してみると分かるが、実験結果と計算結果を比較して、実験結果がどの軌道成分からなっているかを解釈する助けになる計算結果は精度が低くてもよいことがわかる。尊敬する先生方は、このような解釈の助けになる情報を利用して研究を行っている。例えば、細野先生は、講演で理論の話も出されるが、それは現在と比べれば計算資源が少ない時代の話である。その時代でも大きな成果が出ている。しっかりと精度を認識している先生は、不必要な精度で計算すると、どうして高い計算コスト（高い精度）で計算したのかと指摘する（それは当然だと思う）。計算資源は無限ではない。一方、学習のために低い精度と高い精度の双方を計算するのは良い経験になる。そのために積極的に新しい計算手法を利用して計算技術と研究へのセンスを磨くことには大変大きな意義がある。

どのように実験結果と比較して解釈すれば良いかを以下に示す。
・ピークに対する軌道の解釈
　低い精度から初める。明らかに電子構造が違い過ぎるという場合に高い精度の計算をする
・ギャップ近傍にf軌道があってその分裂の程度で熱電性能などの物性が変わる場合
　相対論版で計算する
・遷移金属イオンや希土類イオンの多重項エネルギー準位や吸収スペクトル（UV-vis）
　DVMEで計算する（※ 遷移金属のL2,3-edgeがDVMEで計算できるかはまだ調べていない）
上記以外の場合は、低い精度から初める。明らかに結果がが違い過ぎるという場合に高い精度の計算をする。相対論版は、縮退が解けて、主に価電子帯のピークがわずかにブロードになる（一方、内殻では2つに分裂）という結果になるので、通常の非相対論版DV-Xαで研究の最初にはこれで十分である。
※ プロの計算屋さんなら「計算コストの低い状態から計算して、徐々に高い計算コストにして、目的となる物性を評価できるところで計算を止める」と答えるだろう。
※ 化学系で化学的精度が必要な場合、DFT（構造最適化） > cc-pCVQZ(or cc-pVQZ)以上の基底関数 + CCSD(T)（一点計算）となる。

□ 「もしあなたがレフェリーになったら」を考える
※ 論文を書いている人が知っているかどうかを調べるために指摘されている場合もあるので丁寧に説明するとよい。
　以下のことが分かっていて、この計算だと○○が解釈できるけど、△△についての証明はできていますか？などの質問が来ていれば、本当のプロフェッサーが質問してきていると心の中で感動しながら返信しよう。
・実験結果と計算結果でのエネルギー（横軸）の違いは交換相関ポテンシャルのためであることが常田先生が書かれた書籍に書かれているので、エネルギー（横軸）のズレを指摘してきたレフェリーは第一原理計算（特に、交換相関項の種類や遷移状態（TS）計算について）の勉強が足りないと思った方がよい。相対的な位置関係があっていれば軌道の議論ができるのでそれで十分である。
・吸収スペクトルの強度の違いは交換相関ポテンシャル（具体的には、配置間相互作用（CI）やBSEを導入していないため）や遮蔽効果（XAFSの場合は励起されて空いた内殻を他の電子が埋めやすいかどうか）によるものなので、それについて指摘してきたレフェリーは第一原理計算（特に、CIやBSE, 遮蔽効果）の勉強が足りないと思った方がよい。
・ギャップ幅が実験結果と違うというレフェリーは、XαやLDA, GGAなどの密度汎関数理論による交換相関ポテンシャルが厳密に交換項を計算できてないことを理解していない。これを指摘してきたレフェリーは第一原理計算（特に、交換項と相関項）の勉強（常田先生の書籍を読む）が足りないと思った方がよい。厳密に計算できていなくてもギャップ幅が過小評価されやすいということを理解できていればよい。相対的な評価はできる。傾向が実験結果と合わなければ交換項だということが分かり、他の計算コードでLDA+UでUを変えてUの評価をしてみればよい。
・理論での電子構造の細かなギザギザと実験結果のブロードな結果が合わないと指摘してきたレフェリーは、装置の分解能（ガウス分布）や緩和時間による幅の広がり（ローレンツ分布）によるものが影響してきていることを理解していない。主要な成分を説明できていれば十分。細かなところは高い精度を追求している理論の先生方に任せよう。
・光電子分光スペクトルと理論での強度が違うと指摘してきたレフェリーは、光イオン化断面積の理解が足りない。むしろ生データである電子状態密度分布（DOS）を見せて貰わないと本質的な物性を議論できない。どうしてもPES計算の結果を見せたい場合は、通常のDOS計算、PES計算、実験結果の３つを並べて比較するとよい。
・形成エネルギーや構造最適化の議論ができないと指摘してきたレフェリーは第一原理計算（特に、交換相関項の種類について）の勉強が足りないと思った方がよい。Xαでは精度不足。LDAやGGAなどが必要。他の計算コードを使えばよい。一方、DVXαにSIC補正した(TESDA及びCoulombの)場合は可能。
・クラスターの規模が足りないんじゃないかと指摘してきたレフェリーは第一原理計算の経験が足りないと思った方がよい。別の論文にはもっと大きなクラスターがあるからと指摘する人がいるが、計算資源やコード自体（またはコンパイラ）による制限を知識として持っていない。この場合には、クラスターの大きさを徐々に大きくし、結果が収束に向かっている様子を見せ、結果の解釈が変わってないことを示せばよい。
・精度が足りないのではと指摘してきたレフェリーは研究のセンスを磨いていないと思った方がよい。物性を解釈する本質を明らかにすることを目的とした場合、むやみに高い精度で計算する必要の無いことを説明すればよい。この場合には、クラスターの大きさを徐々に大きくした結果を見せ、結果が収束に向かっている様子を見せ、結果の解釈が変わってないことを示せばよい。
