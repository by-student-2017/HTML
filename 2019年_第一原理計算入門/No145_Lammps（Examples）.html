□ Important information!

・The potential used has a corresponding "units"
  （使用するポテンシャルには対応する"units"がある。exampleの例を参考にrealかatomicかなどを指定する必要がある）
  （Tersoffなどではポテンシャルファイルに記載されていたりする。ポテンシャルの指示に従えばよい）
・In case of "Error: Lost atoms", set "dt" to a value smaller than the current value.
  （"Error : Lost atoms"の場合には、"dt"を現在の値よりもより小さい値にする（原子が周期をまたぎ過ぎることを防ぐ））
・Since the Gear method has good accuracy, "dt" can be made larger than the Verlet method.
  (ギア法は精度が良いのでベルレ法よりも"dt"をより大きくできる)
・The results of “nose” and “scaling” are almost the same.
  （能勢法とスケーリング法は結果がほぼ同じになる）
  （能勢法は数学的な厳密性を求めたり、熱の揺らぎを小さいしたい場合に用いられる）
・The virtual mass of the “nose” method has no physical meaning. It is sufficient if the temperature can be controlled well.
  （能勢法の仮想質量には物理的意味はない。温度を上手く制御できればよい）
・
The temperature is calculated from the kinetic energy. For this reason, when the energy corresponding to the temperature given to the kinetic energy flows to the potential energy, the temperature decreases. This is often seen early in the calculation.
  （温度は運動エネルギーから計算される。そのため、運動エネルギーに与えた温度分のエネルギーが位置エネルギーに流れると温度が下がる。これは計算の初期においてよく見られる）

◇ Note (https://lammps.sandia.gov/doc/fix_nh.html)
  A Nose-Hoover thermostat will not work well for arbitrary values of Tdamp. If Tdamp is too small, the temperature can fluctuate wildly; if it is too large, the temperature will take a very long time to equilibrate. A good choice for many models is a Tdamp of around 100 timesteps. Note that this is NOT the same as 100 time units for most units settings. A simple way to ensure this, is via using an immediate variable expression accessing the thermo property ‘dt’, which is the length of the time step. Example:
  fix 1 all nvt temp 300.0 300.0 $(100.0*dt)

□ VESTAとcif2cellを用いた構造ファイルの作成
1. VESTA (remove symmetry -> P1) > cif
2. cif > cif2cell -p lammps -f  *.cif
※ 上記は atom_style atomic タイプのものとなる

□ VMDを用いた構造ファイルの作成
1. VMD
2. File > New Molecule... 
・MOPAC2016 or Avogadro, VESTA
  > (Filename: [) Browse... > *.xyz (e.g. _opt.xyz)
  > Determine file type: [XYZ] or [Automatically] > Load
・VASP (from VESTA) (for periodic system)
  (need some space, e.g. "    C    H " is good, "    C    H" is bad.)
  > (Filename: [) Browse... > *.vasp
  > Determine file type: [VASP_POSCAR] > Load
3. Graphics > Representations > CPK (in Drawing Method)
  > Name (in Coloring Method)
4. Graphics > Colors...
  > Name (in Categories) > Names [H] > (check Colors)
  ※ if you can see pink color from *.vasp, check *.vasp input file and add a space after the last atom (e.g. change "    C    H" to "    C    H ").
5. Extensions > Tk Console
  > topo writelammpsdata c:/Users/username/Desktop/file.dat full
※ Please change "username" of your PC.
※ full or atomic,  angle, etc

□ 炭化水素の例-1- (MEAM ポテンシャル）
・ NISTのIsobutaneのポテンシャルと入力ファイル
（https://www.ctcms.nist.gov/potentials/system/CH/）に炭化水素のポテンシャルと入力ファイルの例がある。
  Nouranian_CH_library.meam
  Nouranian_CH.meam
  Isobutane.in
  Isobutane.dat
  これらをダウンロードして用いる。
・datは上記のVESTAとOpenbaleを利用したものでよい
・ztc.in (Isobutane.inを参考に書き換える)
----------
#LAMMPS Input file
# Title

clear

# structure settings
units metal
boundary p p p
atom_style full
read_data ztc.dat

# potential settings
pair_style meam
pair_coeff * * Nouranian_CH_library.meam C H Nouranian_CH.meam C H

# output setting
dump           dumpXYZ all xyz 100 ztc_lammps.xyz

# run structure optimization
minimize 1e-25 1e-25 10000 10000

# RDF calculation
reset_timestep 0
compute gofr all rdf 100  1 1  2 2  1 2  # C-C, H-H, C-H
fix rdf all ave/time 100 10 1000 c_gofr[*] file rdf.dat mode vector
run 1000
----------
・ mpiexec -np 1 lmp_mpi -in ztc.lmp
・ gnuplot
・ plot "rdf.dat" u 2:3 w l
・ 出力された xyzファイル（ここでは ztc_lammps.xyz)の中に各サイクルでの構造の結果が記載されている。必要なものだけを残して、Avogadroに入れてxyz形式で出力し、サクラダエディタなどのライティングソフトで元素名を正しいものに置換してVESTAで出力すればよい。
※ 周期構造の場合、周期構造を対称性P1などで作成し、AvogadroからのxyzをimportしてVASP形式で出力すればよい。
※ 原子間距離が近すぎるものがあったり、六員環が大きく歪んでいたり、六員環が壊れていたりする
※ minimize は緩和計算で0 Kと同等のイメージ。必要な設定はポテンシャルで、通常のMDでの計算で必要な質量や時間刻み、アンサンブルの設定は必要ではない。ディフォルトではCG法（共役勾配法）で最小のポンテシャルになる位置に移動する
※ RDFの出力であるrdf.datは[番号, 半径r]の後は[g(r), sum of g(r)]の繰り返し。全体が欲しければ compute gofr all rdf 100 とすればよい。
※ ave/timeの後の数値は 100 * 10 = 1000 でrunの回数とも対応させると動いた

■ 分子動力学でよく使われる計算条件 [MD1]

・総ステップ数： 5000 - 10000 steps
  100000 steps (NEV, NTP)（緩和、セルサイズ算出、本計算）
・時間刻み幅： 0.5 - 1.0 fs
  水素などの軽い原子: 0.1 fs
  固体系: 0.5 - 5 fs
・出力間隔： 100 step 
・温度： 298 K
・圧力： 1 atm = (0.101325 MPa)
・boxの大きさ: 17.62 Å以上
  この値にならない場合は"replicate"でboxを大きくする
・ブックキーピングの球殻の厚さ(dr): 0.2 Å（固体）、0.5 Å（液体）
※ 値を変える場合、[総ステップ数 * 時間刻み幅]となるようにされていたりする
※ MDでは計算開始から数十psの計算結果は評価に使わない。ポテンシャルエネルギーなどで収束をチェック
※ 時間刻み幅（timestepはディフォルトで1.0 fsになっている。ディフォルトは妥当な値）。軽い原子ほど時間刻み幅は短くする。重い原子になるほど時間刻み幅を長くしても精度のよい計算が可能になる（振動周期が長くなるため）
※ NaClなどの溶融塩の計算では、適切なBMHポテンシャルを用いて、atom_style chargeとすれば、read_dataではbondsやangleなどの設定がなくても上手く計算できる
※ SiO2やAl2O3などのガラスの計算では、適切なBuckinghamポテンシャルを用いれば、read_dataにおいてbondsやangle, dihedrals, impropersなどの設定が0でも上手く計算できる

■ 有機物でよく使われる設定（例えば、イオン液体など）

◇ Lennard-Jonesでの入力ファイルの設定 [30]
  (units real を設定の場合）
・atom_style full
  read_dataで指定したファイルに記載されているbondやangleなどの情報を計算に用いる
・lj/cut/coul/long 12.0 12.0
  短距離相互作用にはLennard-Jones（LJ）ポテンシャル
  長距離相互作用のクーロン力はEwald法（またはPPPM法）で計算（dtを大きくとれるようになる）
  LJポテンシャルを12.0[Angstroms]まで計算
  クーロン力は12.0[Angstroms]まで直接計算した後、それ以外のところではEwald法（またはPPPM法）
・mix geometric tail yes
  異種原子間のパラメータを下記のように計算
  -----
  epsilon_ij = sqrt(epsilon_i * epsilon_j)
  sigma_ij = sqrt(sigma_i * sigma_j)
  -----
  mix arithmetic にするとよく知られた"ローレンツ・ベルテロ（Lorentz-Berthelot）則"で計算される
・kspace_style ewald 1.0e-5
  1.0e-5 や 1.0e-6 がよく用いられる。Force（力）の精度を決める値で、この値が小さいほど精度が高くなる
  kspace_style pppm 1.0e-5でも良いが構造によってはewald法より発散しやすい（多少ewald法より計算が早い程度なので最初は"kspace_style ewald 1.0e-5"で計算の後に"kspace_style pppm 1.0e-5"とするのがよい
・bond_style harmonic
  原子内のボンド間のポテンシャルを設定
  harmonicは調和振動子ポテンシャル（最もよく使われる）
  morseはMorseポテンシャル（次によく使われる）
・angle_style harmonic
  原子内の結合角ポテンシャルの設定
  harmonicは調和振動子ポテンシャル（最もよく使われる）
・dihedral_style opls
  二面角ポテンシャルの設定
  opls（オーピーエルエス）などがよく設定される
・improper_style cvff
  cvffなどがよく設定される
  ねじれ角
・fix 1 all shake 0.0001 20 0 b 1 6 10 12
  accuracy tolerance of SHAKE solution: 0.0001
  max # of iterations in each SHAKE solution: 20
  print SHAKE statistics every this many timesteps (0 = never): 0
  b values = one or more bond types: 1 4 7 12
※ この他、"special_bonds lj/coul 0.0 0.0 0.5"などが使われる
※ read_dataにおいてbondsやangle, dihedrals, impropersなどの設定が必要
  イオン液体： E = E(vdW) + E(coulomb) + E(bnd) + E(angle) + E(dihedral)
  希ガス（Arなど）： E = E(vdW)
  溶融塩（NaClなど）： E = E(vdW) + E(coulomb)
  として計算されることが多いことからわかる。
※ カーボンナノチューブでは REBOポテンシャルを用いれば、atom_style atomic でも座屈の計算などが可能

■ Lammps (keyword)

◇ 基本的な設定
・timestep： 1ステップの時間
  unit real -> fs
    [A, fs, Kcal/mol] unit
  unit metal -> ps
    [A, ps, eV] unit
・thermo: MD実行中の情報の出力設定(毎回どのMDの計算回数で結果を出力するかを指定できる)
・thermo_style: customを記述すると、それ以降に出力したい物理量をユーザーが任意にしていできる
・fix (runで実行される系の制御で、nveやnvt, nptなどがある）
  npt: 粒子数(n), 圧力(p), 温度(t)の3つが一定。周囲に水を配置して最適な単位胞を得る（ヒーティングや平衡化と呼ばれる操作）を行うのに使われる。狙いとしては圧力を一定にして水分子を隙間なく配置するために行われる。[4] 密度はnptで計算
  nvt: 粒子数(n), 体積(v), 温度(t)の3つが一定。水の配置が必要ない系や平衡化の後の構造で用いられる
       表面張力をMDでは見ることができる
  nvp: 粒子数(n), 体積(v), 圧力(p)の3つが一定。ヒーティングや平衡化の後のnvtの後に行われる
  nve: 粒子数(n), 体積(v), エネルギー(e)の3つが一定。
       温度ムラを再現したいときや、スパッタ、化学反応のシミュレーションなどで用いられる
       熱伝導率はnveで計算
・minimize: エネルギー最小化 (CG法＝conjugate gradient method = 共役勾配法）
・variable: 変数をユーザーが任意に指定できる
  "variable 変数名 equal 数値"などとすることで${変数名}として利用できる。
・print: 画面に出力
※ NVTアンサンブルの場合、LAMMPSはNose-Hoover法を使用しているので温度緩和は100にすると良いとのこと。tchainは熱浴における振動制御のパラメータでNose-Hoover法の場合デフォルトは3。[30] or (https://lammps.sandia.gov/doc/fix_nh.html)
※ 圧力アンサンブルは、平均二乗変位などの物性値算出には不適切。一方で、ゼロ応力の格子サイズを求めるのには便利であるため用いられる

■ heat capasity at constant pressure
----------
heat capasity at constant pressure for water
(https://lammps.sandia.gov/threads/msg18757.html)

Chris,

You should probably read the prior literature on these methods, as there are lots of inherent pitfalls that you do not want to relearn. For example, I suspect you are using the wrong definition of p. I would use the specified
external pressure, not the instantaneous internal pressure. Also, there is an extensive literature demonstrating that NPT/NPH volume fluctuations do not reliably predict bulk modulus. This will also be an issue for Cp.

A more robust method for obtaining Cv and Cp is to measure <KE+PE> and <T> in NPH and NPT simulations, and then calculate the slope of <KE+PE> versus <T>.

Aidan

◇ packmol.inp [P2]
1. gedit packmol.inp
-----
tolerance 2.0
filetype xyz
output system.xyz

structure solvent.xyz
  number 10000
  inside box 0. 0. 0. 80. 80. 80.
end structure
-----
・tolerance は異なる分子の原子間の最小距離(LAMMPSのunitsによる）。常温常圧なら2.0で良い
・solvent.xyz は溶媒分子の座標ファイル名
・number は溶媒分子の数
・inside box は溶媒分子を詰める領域
