■ Van der Waals interaction (dispersion) (DFTB+ 19.1, manual)

  The Dispersion block controls whether DFTB interactions should be empirically corrected for van der Waals interactions, since DFTB (and SCC-DFTB) does not include these effects. Currently, three different dispersion correction schemes are implemented (for the detailed description of the methods see the following subsections):

・Lennard-Jones
  Dispersion is included via a Lennard-Jones potential between each pair of atoms. The parameters for the potential can either be entered by the user or the program can automatically take the parameters from the Universal Force Field (UFF) 
Example using UFF parameters:
----------
  Dispersion = LennardJones {
    Parameters = UFFParameters {}
  }
----------

・Slater-Kirkwood
  The dispersion interaction between atoms is taken from a Slater-Kirkwood polarisable atomic model 

・DFTD3
  Dispersion is calculated as in the dftd3 code [18, 19] (see section 2.4.10). Modi?cation hydrogen bond interaction strengths (see section 2.4.12).

□ Important information
----------
  The velocity Verlet driver should have a time step on the scale of about 10 times the highest vibrational period in the system. 1 fs is a common choice.
----------
  Fast XL-BOMD is potentially less stable, so should only be used for systems which show good SCC behaviour (molecules without degeneracies or solids with wide band gaps).
----------
  In the special case of the electronic temperature of the system being set to be 0, there is a faster version of this correction which can be used:
  ForceEvaluation = DynamicsT0
---------- (Simulated annealing)
  The first example anneals away a Stone-Wales defect in a graphene sheet. This is acheived by heating the system up to a high (5000K) temperature where C-C bonds start to break, holding it at these temperatures for a while and then cooling to a low temperature. The high temperature is above the usual disociation temperature of graphene, but since only a relatively short (2.4 ps) time is computed, using a higher temperature accelerates the annealing.

Turned off mulliken analysis in non-scc runs produces garbage #447

When the Mulliken analysis is explicitely turned of in non-scc runs, the resulting energy only contains the repulsive contribution. The electronic part is set to zero, resulting in invalid results.
