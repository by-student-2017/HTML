■ Coding [21]
(Edit: Jun/27/2021)

◇ fundamental minimal structure
--------------------
class class_name(BaseXXX):
  def __init__(self, yyy):
    super().__init__(zzz)
      self.aaa = bbb(ccc)
      self.ddd = ['eee','fff']

  def featurize(self, x, ggg):
    tmp = self.aaa.transform(x)
    output = self.hhh(tmp)
    return self.scaler.inverse_transform(output)

  @jjj
  def feature_labels(self):
    return self.ccc
--------------------
◇ Note (for exmaple,)
・dd = transform(data['SMILES']): transform input (SMILES) to descriptor dataframe.
・inverse_transform(dd): transform descriptor dataframe to input (SMILES).

◇ Basic components
・converter (descriptor)
  input (S) -> property (Y)
  library (S) -> property (Y) 
・generator (modifier) = P(S|Y)
  property (Y) -> input (S)
・evaluator (property)
◇ Note (for example,)
・S = Structure (fingerprint).

◇ Inverse design
・Generator (Modifier) <--> Likelihood (Descriptor, Estimator)

◇ coding rules (more detail)
(If anything, the design mainly uses scikit-learn.)
・BaseFeaturizer
  (BaseCompositionFeaturizer, CrystalGrraphFeaturizer, FrozenFeaturizer, RDKitFP, MACCS, etc)
  Input: list-like(=list,numpy.array, pandas.series)/pandas.DataFrame
  Output: numpy.array/pandas.DataFrame/custom
  Necessary coding: def @jjj (featurize and fature_labels)
    to use default parallelization setup. write featurize as a single input to list-like output descriptor calculator.
    to use custom parallelization, implement it in featurize and fix n_jobs=0 in __init__
    feature_labels should return list of strings.
・BaseLogLikelihood
  (GaussianLogLikelihood)
  Input: custom (recommend pandas.DataFrame -> matching BaseFeaturizer)
  Output: pandas.DataFrame
  Necessary coding: def log_likelihood()
    optional input targets is for defining desing target.
    column names of outputs are target propoerties.
  Optional coding: def fit()
    this function is for any kind of model training involved in the log_likelihood
・BaseDescriptor/BaseLogLikelihoodSet
  (Fingerprints, Compositons, Structures)
  Input: same as corresponding BaseClass/pandas.DataFrame
  Output: same as corresponding BaseClass
  Necessary coding: class class_name(BaseXXX):
・BaseProposal
  (NGram, ReactantPool)
  Input: list-like(=list,numpy.array, pandas.series)/pandas.DataFrame
  Output: same as input
  Necessary coding: def proposal
    produce new candidastes (with or without input candidates)
    input/output type should be designed to match with descriptors for the problem of interest.
  Optional coding: def fit()
    this function is for any kind of model training involved in the proposal
・BaseSMC
  (IQSPR, IQSPR4DF, BaseResample)
  Input: match with BaseProposal
  Output: same as input + (options)
  Necessary coding: def unique and resample
    use of this basee class is only intended for more advance users with special needs.
  Optional coding: def __call__
    changing this function is not recommended except for more advance users with special needs.
