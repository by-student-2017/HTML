■ xtb v.6.2.3 (ubuntu 18.04 on windows10 (64 bit)) (Edit: 3/Jun/2020)
  (https://www.chemie.uni-bonn.de/pctc/mulliken-center/software/xtb/xtb)

□ download and unpack xtb binary
(https://github.com/grimme-lab/xtb/tags)
1. wget https://github.com/grimme-lab/xtb/releases/download/v6.2.3/xtb-200316.tar.xz
2. tar xf xtb-2*.tar.xz

◇ environment settings
1. gedit ~/.bashrc
#----------
# xtb settings
export XTBHOME=$HOME/xtb_6.2.3
export XTBPATH=${XTBHOME}/share/xtb:${XTBHOME}:${HOME}
export MANPATH=${MANPATH}:${XTBHOME}/share/man
export PATH=${PATH}:${XTBHOME}/bin
export LD_LIBRARY_PATH=${LD_LIBRARY_PATH}:${XTBHOME}/lib
export PYTHONPATH=${PYTHONPATH}:${XTBHOME}/python
#----------
※ 上記を ~/.bashrc の最後に書き加えればよい。(Add the above to the end of "~ / .bashrc".)
2. bash
----------
※ xtb-6.3.1 case (change part)
  export XTBHOME=$HOME/xtb-6.3.1
  export PATH=${PATH}:${XTBHOME}/bin:${XTBPATH}

◇ version check
1. xtb --version

◇ help
1. xtb -h
2. xtb --help

◇ parallel calculation settings before run
1. (ulimit -s unlimited)
2. export OMP_STACKSIZE=1G
3. export OMP_NUM_THREADS=2,1
  (above command for 2 cpus)
  (e.g., "export OMP_NUM_THREADS=4,1" for 4 cpus)
  (export OMP_MAX_ACTIVE_LEVELS=1) (default 4)
4. export MKL_NUM_THREADS=2
  (above command for 2 cpus)
  (e.g., "export MKL_NUM_THREADS=4" for 4 cpus)

□ Singlepoint Calculation

◇ H2O
1. cat << EOF > coord
\$coord
 0.00000000000000      0.00000000000000     -0.73578586109551      o
 1.44183152868459      0.00000000000000      0.36789293054775      h
-1.44183152868459      0.00000000000000      0.36789293054775      h
\$end
EOF
2. xtb coord
  (e.g., 4 cups case: xtb coord --parallel=4)
  (You can get "charge")

◇ MgO (periodic bundary and fractional coordinates)
1. cat << EOF > coord
\$coord frac
    0.00      0.00      0.00      mg
    0.50      0.50      0.50      o
\$periodic 3
\$cell
 5.798338236 5.798338236 5.798338236 60. 60. 60.
\$end
EOF
2. xtb coord --periodic
  (e.g., 4 cups case: xtb coord --parallel=4)
--------------------
□ Structure optimization

◇ TiO2 (periodic bundary and fractional coordinates)
1. *.cif -> VESTA -> tio2.vasp (Cartesian)
2. mv tio2.vasp tio2.poscar
3. xtb tio2.poscar --periodic --opt
  (e.g., add other options: xtb tio2.poscar --periodic --opt --iteration=250 --cycles=250 -acc=1.0 --parallel=4)
4. jmol xtbopt.poscar
5. jmol xtbopt.log

◇ TiO2 (periodic bundary and fractional coordinates with xcontrol)
1. *.cif -> VESTA -> tio2.vasp (Cartesian)
2. mv tio2.vasp tio2.poscar
3. cat << EOF > xcontrol
\$opt
    engine=inertial
\$end
EOF
4. xtb tio2.poscar --periodic --opt --input xcontrol
  (e.g., add other options: xtb tio2.poscar --periodic --opt --input xcontrol --iteration=250 --cycles=250 -acc=1.0 --parallel=4)
4. jmol xtbopt.poscar
5. jmol xtbopt.log

◇ TiO2 (cluster)
1. *.cif -> VESTA -> tio2.xyz
2. xtb tio2.xyz --opt
  (e.g., 4 cups case: xtb tio2.xyz --opt --parallel=4)
3. jmol xtbopt.xyz
4. jmol xtbopt.log

※ PDB file: *.cif -> VESTA -> tio2.pdb -> xtb tio2.pdb (failed in my case)
※ 構造のファイル（coord, xyz, poscar）は xtb の直後か一番最後に記述する
※ *.poscar では --hess が上手くいかなかった

□ Vibration (IR and Raman spectrum and hessian)

◇ H2O
1. cat << EOF > coord
\$coord
 0.00000000000000      0.00000000000000     -0.73578586109551      o
 1.44183152868459      0.00000000000000      0.36789293054775      h
-1.44183152868459      0.00000000000000      0.36789293054775      h
\$end
EOF
2. xtb coord --opt --parallel=2
3. xtb xtbopt.coord --hess --parallel=2
(above 2.+3. case: xtb coord --opt --parallel=2 && xtb xtbopt.coord --hess --parallel=2)
(You can get "vibspectrum", "hessian" and "g98.out.")
--------------------
□ Molecular dynamics (MD)

◇ Geometory optimization -> MD
1. xtb coord --omd
   (xtb case.poscar --periodic --omd) 

◇ Restart MD
1. cat << EOF > restart.inp
\$md
  restart=true
EOF
2. cp scoord.XX coord_last
  (XX=number, e.g., last number)
3. mkdir 1st_run
  mv scoord.* ./1st_run
  mv xtb.trj ./1st_run
4. xtb coord_last --input restart.inp --md
  (xtb coord_last --periodic --input restart.inp --md) 

◇ TiO2 (periodic bundary and fractional coordinates)
  (Constrained MD/MTD simulations)
1. *.cif -> VESTA -> tio2.vasp (Cartesian)
2. mv tio2.vasp tio2.poscar
3. cat << EOF > metadyn.inp
\$md
   time=10
   step=1
   temp=298
\$end
\$metadyn
   atoms: 1-3
   save=10
   kpush=0.02
   alp=1.2
\$end
EOF
4. xtb tio2.poscar --md --input metadyn.in --periodic
  (e.g., 4 cups case: xtb tio2.poscar --md --input metadyn.in --periodic --parallel=4)
5. jmol xtb.trj
※ restart する場合には $md に restart=true を追加する
--------------------
□ restart
1. xtb coord --periodic --md
2. xtb coord --periodic --restart --grad
  (you can get gradient and gradlatt)
3. xtb coord --periodic --md --restart

□ output file including error
xtb Al_3x3x3.xyz --periodic 2>&1 | tee Al_3x3x3.out
