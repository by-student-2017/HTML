□ インストールとテスト
1. https://www.szfki.hu/~nphys/rmc++/downloads.html での「Windows package」から下の2つをダウンロードする
  ・ Download the latest (Version 1.4) RMC_POT++ Windows consecutive ready to use package (compressed with Winzip 5,163 kB). 
　・ test run 
  ※ parallel版（並列版）は pthreadvce2.dll がないとエラーが出る。
2. LHaplusなどで解凍する
3. 解凍した test_run で dummy に入る > Shift+右クリック > コマンドウィンドウをここで開く
　※ test_run の readme.txt を読んで、それに従えばよい
4. 解凍した Ready_RMC_POT_win_1.4 の中にある RMC_POT_i64_1.4.exe を コマンドウィンドウにドラッグ&ドロップ > Enter
5. where the run information should be read from-> AsSeI

□ 入力ファイル
・ 標準的な入力ファイルは、*.cfg, *.cat, PPCF's（実験データ）の3つ
・ 入力ファイルの*.cfgと*.datはRMCAとほぼ同じ
・ *.cfg: The text type configuration file containing the coordinates.
　原子座標のデータが入る
・ *.dat: INPUT: the data file containing the parameters of the run.
　入力はいくつかの部分に分かれている（行番号はマニュアルに記載の値）
　1-10行目： 繰り返し計算の設定
　11-16行目： g(r)データの読み取り設定
　17-25行目： 中性子散乱データの読み取り設定
　26-33行目： X線散乱データの読み取り設定
　34-43行目： EXAFSデータの読み取り設定
　55-60行目： レナード・ジョーンズ（LJ）ポテンシャルの設定
・PPCF's（実験データ）
　*.gr: g(r)データ
　*.sq: 中性子散乱
　*.fq: X線散乱（x軸はq値）
　*.ek: EXAFS（x軸はk値、y軸はE(k)）
　　EXAFSでは exafs_coeff.dat として、coefficient file for EXAFS のデータ（exafs_coeff.dat）が必要（FEFFで作成する）
　----------
　1行目： データ数
　2行目： コメント
　3行目以降：データ（3行目以降に1行目に書いたデータ数だけ記述）
※ マニュアル（https://www.szfki.hu/~nphys/rmc++/RMC_POT_user_guide.pdf）に書いてある

◇ 実際の入力ファイルの作り方
1. test_run の dummy の *.cfg と *.dat ファイルを書き換える
　分子の場合は、ccl4_atとccl4_molの*.fncと*.cusファイルを参考にする
2. cfg （変更点）
　0 0 0 moves generated, tried, accepted ← RMCでの計算の履歴
　----------
　1200 molecules of all types の数が下の例で示す（type1-3）の合計値と一致する
　510 molecules of type  1
　510 molecules of type  2
　180 molecules of type  3
　※ 「1 atomic sites」の次の行は 0.0 0.0 0.0 でよいように思える
　----------
　F (Box is not truncated octahedral)　← 計算する箱の形状の設定
  ----------
　type1の原子から順番にx, y, zを記述していく。
  （上記の例だと、最初から510個までAs、その次の510個のデータがSe、その次の180個がIの原子座標となる）
3. *.dat （変更点）
・ マニュアルのp.27に設定についての説明が記載されている
・ 表中のアンダーバー&イタリックの記述はオプションで記述しなくてもよい
　----------
　0.0721  ! number density (Angstrom^-3)　
　　Å-3単位。*.cfgと一致しなければ*.cfgでの箱のサイズが変更される
　----------
　3.0　! cut offs　←closet distance (Angstrom)
　　二元系AB： A-A A-B B-B （例えばSiO2なら 2.9 1.5 2.4 )
　　三元系ABC： A-A A-B A-C B-B B-C C-C
　　N元系ではN(N+1)/2を同様に記述する
　　想定されるそれぞれの原子間距離の82程度（±0.5%）を設定
　　Cuは1.8、Arは3.0、Hgは2.6が設定されている
　----------
　0.1　! max moves
　　基本的に0.1を選択。SiO2では0.25、Hgでは0.25、Cuでは0.3
　----------
　0.1　! r spacing (Angstrom)
　　基本的に0.1を選択。HgやSiO2では0.05
　　（省略可能）spacing local inv. のオプションは0.05
　----------
　.true.  ! whether to use moveout option
　　カットオフに近い原子を優先的に動かすかどうかの指定
　　CuやAr、Hg、SiO2、CCl4は.false.を指定。AsSeIは.true.を指定
　----------
　0 5 ! no of configurations to collect, frequency for collecting
　　タイムリミット後の配置の修正回数（マニュアルの英語が理解できない）
　　基本的に0を設定
　　frequency for collecting は2-5が設定される。オプションなので記述しないこともある。SiO2では記述さていない。
　----------
　500  ! step for printing
　　500から9000程度が指定される
　　CCl4は500、Arは1000、Cuは5000、AsSeIは4000、HgやSiO2は9000程度
　----------
　50 10  ! time limit, step for saving
　　シミュレーション時間（分）、保存の間隔（分）
　　シミュレーション時間は 6-60 程度（50程度の設定が多い）
　　保存は 2-10 程度（10程度の設定が多い）
　　これらの設定時間が短いのはCCl4
　　Cuは両方とも 100000 と非常に大きい値が設定されている
　----------
　0 1 0 0   ! no. of g(r), neutron, xray and exafs expts
　　g(r)、中性子、X線、EXAFSの4つに対応 （5つ目はオプションでLEAD SERIES INDEX）
　　中性子散乱での設定 0 1 0 0
　----------
　21 31  ! range of histogram bin points to use for EXAFS (r space)
　3  ! chi(k) weighting ←E(k)=knΧ(k)での重みn。nは1-3の値
　　EXAFSのみ記載。g(r)や中性子、X線は記載しない
　----------
　Ar.sq など読み込む実験データのファイル名を記載（.sqや.fqではなく.datでも動作する）
　----------
　1 100   ! range of points
　　Q点の読み取り範囲（Q値ではない）
　　g(r)や中性子、X線で記述が必要
　　EXAFSはk点なので「 ! no. of k data points (starting with 1)」で記載する
　----------
　0.00  ! constant to substract
　　基本的に0.00 を設定
　　g(r)、中性子、X線で記載。EXAFSは記載しない
　----------
　1 ! type of absorbing particle
　　EXAFSのみ記載。基本的に1を指定
　　g(r)や中性子、X線は記載しない
　----------
　0.0218  0.2520  0.7262 ! partial coefficients coherent(1-1, 1-2, 2-2...)
　　g(r)や中性子で記述が必要
　　X線は読み取る実験データ（*.fq）に記述される
　　EXAFSは「exafs_coeff.dat ! coefficient file for EXAFS」で記載
　　二元系AB： A-A A-B B-B
　　三元系ABC： A-A A-B A-C B-B B-C C-C
　　N元系ではN(N+1)/2を同様に記述する
　　係数の合計値は1になる
　　これらの partial coefs は下に示す式で計算する
　　
　　
　　ここでciとcjはモル比、biとbjは中性子では散乱長、X線では原子散乱因子f(Q)である。中性子の散乱長はQに依存しない定数と扱えるため計算し易い（上の式ではバーが上に付いているので同位体比率での平均を用いる。文献[19,20]のBc_realやNISTのデータのCoh bを用いればよい）。一方、X線の原子散乱因子はQに依存するため読み込む実験データに記述される（そのためX線では係数（partial coefficients）の記述は無い）
　----------
　0.02　 ! standard deviation
　　Χ2 = Σ[理論S(q)-実験S(q)]2/σ2でのσ
　　中性子の場合： ArやCCl4では0.001。CuやAsSeIでは0.01
　　X線の場合： AsSeIでは0.005
　　EXAFSの場合： AsSeIでは0.000008 (マニュアルでは「-0.0005 ! weight parameter」なのでtest_runの「! standard deviation」の記述は誤りか？ ソースコードでの確認が必要)
　----------
　.false.　! whether to vary amplitudes (0-false, 1-true)
　.false.  ! whether to vary constant (0-false, 1-true)
　.false.  ! whether to vary linear (0-false, 1-true)
　.false.  ! whether to vary square (0-false, 1-true)
　　測定によるエラーを修正するための規格化をするかどうかの指定
　　基本的に1（または、.true.）でよい
　----------
　.false.  ! whether to use a potential
　　0： LJポテンシャルを用いない（0 = .false.）
　　1： C6 and CN is given for LJ
　　2： sigma nad epsilon is given, arithmetic average （σが相加平均）
　　3： sigma and epsilon is given, geometric average (相乗平均)
　　HgやCCl4、AsSeI、SiO2などは0に設定してLJポテンシャルを用いていない
　　LJポテンシャルについては、左欄の「逆モンテカルロ法（RMC）」での[■ The interaction function]を参照してください。
　　1-3を指定した場合のLJに関係するパラメータの設定はマニュアルをご確認ください
　　※ different GROMACS typeについてはマニュアルではよく分からないのでソースコードを参照した（RunParams.cpp の970行近くに記載されている）
　※ RMCAのCuの入力ファイルはここで.false.にして終了。恐らく上記までの入力がRMC計算で最低限必要なのだろう
　----------（ソースコードで確認していないので誤りがあるかも）
　2 1 2  2 1　 ! no of coordination constraints, (省略）
　　次の行から書かれる配置の制限の数 隣接原子の数 各制限の数 1
　2  1   2.1  2.7 2   3     0.4  0.6 0.00015 0.00025 ! central type,（省略）
　　中心の原子種 その周りの原子種  距離2.1から 距離2.7から　2まで 3 まで 周りに0.4個配置 周りに0.6個配置 標準偏差σ 標準偏差σ
　3  1   2    2.4 3.0 2.95 3.5  2   1.0        0.00025 ! central type,（省略）
　　（上と読み方は同じ）
　※ CCl4では指定されていないが、他の化合物では指定されている
　----------
　2　! no of average coordination constraints
　　次の行から書かれる平均的な配置の制限の数（ここではSiO2を想定）
　1 2 0. 1.8 4 1. 0.00001
　　中心の原子種 その周りの原子種  距離0から 1.8まで 平均の配位数4 標準偏差σ
　2 1 0. 1.8 2 1. 0.00001
　　中心の原子種 その周りの原子種  距離0から 1.8まで 平均の配位数2 標準偏差σ
　※ CCl4では指定されていないが、他の化合物では指定されている
　----------
　0    ! FNC switch
　　FNC = 拘束条件である Fixed neighbours constraints の設定
　　1-3を指定すると*.fncが必要になる。古いRMCAでは実装されていないため、上手く計算できないときに導入を検討すればよいと思われる。CCl4ではこのオプションは1を指定している。
　----------（基本的にあまり変更のないもの）
　0  ! initial bin shift
　1.414  ! xmax used in the run (Angstrom) 
　1  ! number of atoms moved in a single move
　1  ! size of the history buffer (0->no history record, 200 is good)
　　Arは1, CCl4は5、AsSeIは100
　5  ! number of savings between each history buffering
　0  ! indicator of custom move (0:no, 1:yes)
　1    ! whether to load the histogram form file, if possible (0: no, 1:yes)
　5  ! maximum number of atoms in a gridcell
　----------
　0.1 1 0 0   ! fraction of swaps, ntypes*(ntypes-1)/2 0 (not allowed) or 1 (allowed) for the possible mixed partial (in order 1-2, 1-3,... 2-3...) 
　　ArやCCl4では 0 1 0 0
　　1つ目の数値はスワップ（位置が入れ替わり）する割合を示す（スワップしない0からスワップのみの1までの値をとる）
　　2つ目以降の数値はその組み合わせでスワップを許さない（0）と許す（1）の指定。これはN(N+1)/2個が記述される。ここでNは原子の種類。記述は2元系ABなら A-A A-B B-B となる。3元系ABCなら A-A A-B A-C B-B B-C C-Cに対しての記述となる。
　----------
　2  ! NUMBER OF THREADS TO USE
　　並列計算での並列数（スレッド並列）の指定
　　test_runの入力ファイルでは1か2が記載されている
　　CCl4はここで入力が終了
　----------
　2 -0.1 -0.2 1 0 0.03 0.06 0.1 ! local inv: number of intervals, sigmas, mode (0:bin, 1:distance-based), n_loc_interval+1 loc_ratios
　　Arでは-0.7  ! LOC INV SIGMA
　　マニュアルにはヒストグラムとbin（おそらく値の範囲）、インターバル（おそらく間隔）が関連事項として記載されているので、ヒストグラムの分布の形状に制限を加えているのだろう。
　　条件を指定する数（Nloc） sigma（Nlocの数だけ入力） モード 比率（Nloc+1の数だけ入力）
　　マニュアルの記述と比較するとtest_runの入力は誤っているのでは？ （コード（RunParams.cpp）では679行近辺からloc_at_ratioがnlocint+1でNloc+1要求されている）。
　----------
　多くの場合、ここまでで入力ファイルの記述は終了

□ オプションの入力ファイル
◇ *.fnc
・ 1行目： コメント
・ 2行目： コメント
・ 3行目： コメント
・ 4行目： 最小と最大の距離の指定の数
・ 5行目： 最小距離（4行目に指定した数だけ入力）
・ 6行目： 最大距離（4行目に指定した数だけ入力）
・ 7行目： 空白
・ 8行目： 原子の数
・ 9行目： 空白
　（次の行から三行ずつ組みで原子の数分だけ記述）
・ 10行目： 原子の番号（各原子に付けられる番号。原子番号ではない）　制限の数
・ 12行目： 10行目で記載した原子を中心として、それに対して距離を制限する原子の番号
　（各原子に付けられる番号。原子番号ではない）
・ 13行目： 制限のタイプの番号
　（4-5行目で指定した距離の制限の番号。4-5行目は左から上下が組みになって番号が1, 2, 3, ... となる）

■ RMC++用のAuxiliary programs
　入力ファイルの補助や出力ファイルの変換などのプログラムがある。FortranやC言語で書かれている。

◇ Crystal
1. Linux用のファイルを開いてダウンロードし、解凍する
2. コンパイル
　gfortran crystal.f90 -o crystal.exe

◇ convert_gromacs
（RMC->Gromacs、Gromacs->RMCの両方の変換が可能）
1. ダウンロードし、解凍する
2. コマンドプロンプトを開いてtestに入る
3. convert_gromacs.exe
4. BMtMe_opls_s.dat
　Gromacs用のファイル*.groが作成される
　※ Gromacsのファイル*.groはVESTAでは開けない。gromacs用のguiで開く。Gromacs-guiか？
　※ *.groの読み取り：VMDは失敗。Rasmolも失敗。*.groを描画するソフトはない？
5. BMtMe_opls_s.dat の1行目を0 1 から 1 1 にする
6. convert_grom
7. BMtMe_opls_s.dat
　*.cfgが作成される
※ 将来的には、Gromacsとの連携まで到達したいが、RMCの専念業務ではないのでどうなるかはわからない。

◇ convert_CMconfigs
1. *.cfg を用意する
　test_runのccl4_molをここでは用いて説明する。
2. 1.par を書き換える
　----------
　ccl4_mol.cfg
　1  ! number of config
　2  ! number of used types
　1  ! mode option, (in case of option 4 index of the perpendicular axis(x:0, y:1,z:2),min, max reduced coord)
　C  2048  ! Atom chemical symbol, number of atoms to use for each type
　Cl 8192   ! Atom chemical symbol, number of atoms to use for each type
　----------
　mode option は1がディフォルトとなる
　※ *.cfgに原子の数が書かれているのでそれをコピー&ペーストする。Cなら「2048 molecules of type  1」と書かれている
3. convert_CMconfigs.exe
4. 1.par と入力
5. *.ctmx が出力される
6. VESTAに*.ctmxを入れる
　 Edit > Edit Data > Unit Cell... > Lattice parameters のa, b, cに、*.cfgの「F (box is cubic)  Defining vectors are:」の値を入力する。Fタイプなのですべて同じ値でよい。ccl4_mol.cfgでは 34.235218 とあるので、a, b, cにこの値を入力　> OK
※ ccl4_mol.cfgで、xが負の値であれば、boxのa軸の長さである34.235218を用いて、34.235218 * ( 1 + x ) とすればccl4_mol.ctmxと同じ値が得られる。bやc軸方向も同様。

※ exafs_coeff_beta1 はK-edgeについてのみ計算するが、自動でfeff.inpを作成する。ただし、短距離でエラーが出るので、まだ自動で作成されるfeff.inpのパラメータが適切ではない。使い方は、chmod +x exafs_coeff_beta1 として、実行ファイルに指定し、
　exafs_coeff_beta1 原子1 原子2

　例えば、exafs_coeff_beta1 Ge Te とすると、RMC++のEXAFSのマニュアルに書かれている形に近くなるが、feff.inpの詳細なパラメータの設定方法が書かれていないので上手く行っていない（苦労している）。これらのcshのスクリプトは書き換えたり有効活用して頂ければ幸いだ。適切なfeff.inpを作る為に feff8l を勉強しなければいけないな.......。
※ exafs_coeff_beta2 はFEFF8のマニュアルを読んで可能な限り最適化させたがまだ上手く計算できない。そこで、chiが計算できなかったときは、0.0 とするようにした（サンプルの「test_run」にある「as_edge_bs.dat」も 0.0 となっていたため）。使い方は、
  chmod +x exafs_coeff_beta2
  exafs_coeff_beta2 原子1 原子2
